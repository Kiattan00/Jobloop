# 快速验证指南：JobLoop P0

## 前置条件

- 依赖已安装
- 当前规格为 `specs/001-jobloop-p0/spec.md`
- 视觉风格基准为 `docs/JOBLOOP_VISUAL_STYLE.md`

## 启动方式

```powershell
npm run dev
```

打开本地开发地址后，按新版 P0 页面进入验证。若默认端口被占用，可使用项目现有开发方式切换端口。

当前页面路径：

- `/`：JobLoop P0 首页入口
- `/resumes`：简历库
- `/resumes/generate`：基础简历生成
- `/analyses`：岗位分析工作台
- `/analyses/[jobId]`：单岗位分析详情

## 环境变量

当前实现使用 OpenRouter 的 OpenAI-compatible 服务端调用。运行前需准备：

```bash
OPENROUTER_API_KEY=...
OPENROUTER_MODEL=deepseek/deepseek-chat
OPENROUTER_SITE_URL=http://localhost:3001
OPENROUTER_APP_NAME=JobLoop
```

## 核心路径验证

### 1. 原始简历与基础版本

1. 进入基础简历生成页。
2. 粘贴一份原始简历。
3. 触发基础简历版本生成。
4. 确认页面出现 2-3 个基础简历版本草稿。
5. 保存至少一个版本到简历库。

**预期结果**：简历库展示已保存版本，包含名称、适用方向、改写重点和操作入口。

### 2. 批量 JD 输入

1. 进入批量 JD 输入页。
2. 粘贴至少 3 个 JD。
3. 为其中一个 JD 填写公司补充信息。
4. 确认“联网补充公司信息 Beta / Coming soon”只作为占位入口出现。
5. 触发批量分析。

**预期结果**：每个 JD 独立生成岗位分析结果；未填写公司补充信息的 JD 不会阻塞分析。

### 3. 岗位分析总览

1. 进入岗位分析总览页。
2. 查看至少 3 个岗位卡片或表格行。
3. 比较推荐简历、匹配分、是否值得投、是否需要微调和主要风险。

**预期结果**：用户能在 10 秒内识别最值得优先投递的岗位。

### 4. 单岗位详情

1. 从岗位分析总览点击某个岗位的“查看详情”。
2. 查看完整分析。
3. 复制或阅读打招呼话术。
4. 查看面试准备建议。

**预期结果**：详情页展示匹配分、岗位信息、结论、关键优势、风险缺口、建议行动、打招呼话术和面试准备建议。

### 5. 岗位微调

1. 在单岗位详情页触发微调生成。
2. 查看微调建议或微调版简历正文。
3. 返回详情页或简历库查看关联关系。

**预期结果**：微调内容能看出来源岗位和来源基础简历版本，且不会覆盖原基础简历版本。

### 6. 刷新恢复

1. 刷新浏览器页面。
2. 回到简历库、岗位分析总览和单岗位详情。

**预期结果**：已保存的原始简历、基础简历版本、JD、岗位分析结果和微调内容仍可找回。

## 质量验证

```powershell
npm run quality
```

**预期结果**：格式化、Biome safe fix 和 TypeScript 检查通过。

## 范围检查

- 页面不应出现真实联网搜索结果。
- 页面不应出现 Chrome 插件流程。
- 页面不应出现批量自动投递。
- 页面不应恢复旧的完整求职 CRM 状态流转。
- 页面不应要求面试后反馈录入才能完成 P0 主路径。
