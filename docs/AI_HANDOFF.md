# AI_HANDOFF — JobLoop

## 项目标识

- 目录名：JobLoop
- 名称大小写固定为：JobLoop

## 一句话描述

JobLoop 是一个面向多岗位投递求职者的 AI 工作台。它将简历、JD、面试反馈与岗位进展沉
淀为持续更新的岗位档案，并基于这些上下文生成匹配分析、面试复盘与下一步准备建议。

## 核心闭环

1. 新增岗位
2. 粘贴 JD / 简历
3. AI 生成匹配分析
4. 保存岗位档案
5. 录入面试反馈
6. AI 生成复盘与下一步建议
7. 更新岗位状态

## P0 页面

1. 总览仪表盘
2. 新增岗位
3. 岗位档案页
4. 面试反馈录入页
5. 分析报告页

## P0 LLM 调用

1. JD + 简历匹配分析与结构化提取
2. 面试反馈复盘
3. 下一步准备建议
4. 可选的模拟面试问题生成

## P0 数据表

1. `resumes`
2. `jobs`
3. `analyses`
4. `interview_events`
5. `ai_outputs`

## 明确不在范围内

- 批量自动投递
- 全网抓取岗位
- 复杂 Agent
- 对所有岗位自动深度分析
- 复杂 Word/PDF 导出
- P0 阶段不做公司全量情报研究，可放入 P1 讨论

## 产品原则

P0 必须体现“反馈闭环”，而不是“一次性 AI 报告生成器”。
每一条 AI 输出都必须绑定岗位档案，并帮助用户决定下一步行动。

## 建议实现顺序

1. 确认项目技术栈与包管理器
2. 建立 P0 数据表对应的本地持久化或数据库结构
3. 完成仪表盘与新增岗位页面
4. 完成岗位档案页
5. 接入 JD + 简历结构化提取与匹配分析
6. 将分析结果保存到 `analyses` 与 `ai_outputs`
7. 完成面试反馈录入
8. 生成面试复盘与下一步建议
9. 补充岗位状态更新与简单报告页

## 模型策略

- JD / 简历提取优先使用稳定的结构化输出模型
- 面试反馈复盘与下一步规划可使用更适合长文本推理的模型
- Prompt 设计保持 schema 驱动
- P0 不构建复杂自治 Agent

## 验证清单

- 新增一个岗位
- 粘贴 JD 和简历
- 生成匹配分析
- 保存岗位档案
- 录入面试反馈
- 生成复盘与下一步建议
- 刷新后确认所有记录仍然存在
