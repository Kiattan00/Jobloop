# JobLoop 视觉风格说明

## 目的

本文档用于保留 JobLoop 当前过程稿中已经认可的视觉方向，供后续按新版 P0 规格重做页面和组件时参考。当前已有的 Dashboard、岗位档案、反馈、报告等页面原型可以作为视觉参考，但不再决定新版 P0 的页面数量和信息架构。

新版 P0 页面以 `specs/001-jobloop-p0/spec.md` 为准：简历库、基础简历生成页、批量 JD 输入页、岗位分析总览页、单岗位分析详情页。

## 品牌气质

- JobLoop 应呈现“图书馆档案工作台”的感觉：冷静、专业、可检索、可比较、能沉淀。
- 产品不是营销落地页，也不是聊天机器人首屏；第一眼应像一个正在工作的求职决策台。
- 视觉表达应服务于批量比较、快速筛选、单岗位深读和下一步行动。
- `JOBLOOP` 必须作为稳定品牌信号出现，字号和位置应明显高于普通后台 logo。

## 视觉母版

- 背景采用图书馆或资料馆场景，暗示“档案、检索、准备、知识整理”。
- 背景必须经过暗化、模糊或蒙版处理，保证文字和数据可读。
- 主界面采用半透明玻璃面板，让背景可见但不干扰阅读。
- 页面主体以黑白灰为基础，青色只用于 AI、匹配分、当前状态、关键行动和阶段进度。
- 页面应保持轻量层级，不使用左侧竖栏作为主导航。

## 色彩原则

- 主色：黑、白、灰，用于文本、面板、分隔、基础按钮和信息层级。
- 强调色：天青色或青蓝色，用于匹配分、AI 结果、推荐项、当前步骤、主要行动。
- 风险色：谨慎使用暖色或红色，只用于“不推荐”“高风险”“缺失严重”等判断。
- 禁止大面积紫色、彩虹渐变、娱乐化高饱和配色。
- 禁止把青色当作大面积背景色；它应该是判断与行动的提示色。

## 玻璃界面

- 主容器可以比内容卡片更透明，让背景透出。
- 内容卡片透明度应略高于主容器，以保证文字可读。
- 玻璃层应有轻微边框、模糊和柔和阴影，避免纯白不透明大卡片。
- 圆角应稳定克制，体现工具感；不要为了“柔和”把所有控件做成过度圆润的胶囊。
- 面板内部要有足够留白，但信息密度应适合工作台，不要变成展示海报。

## 信息层级

- 一级：`JOBLOOP`、匹配分、是否值得投、推荐简历等决策信息。
- 二级：页面标题、模块标题、岗位名称、简历版本名称、主要风险。
- 三级：英文辅助标签、说明文本、来源链接、补充信息、次要状态。
- 行动：主要按钮用于生成、查看详情、生成微调版、标记投递；次要按钮用于复制、下载、返回。

## 页面模式

- **简历库**：像可复用素材库，突出不同基础简历版本的定位、适用方向和操作入口。
- **基础简历生成页**：突出“原始简历 → AI 生成草稿 → 用户确认保存”的转换过程。
- **批量 JD 输入页**：支持多个 JD 高效录入，公司补充信息与联网占位入口保持轻量。
- **岗位分析总览页**：使用卡片或表格做多个岗位的决策级比较，重点是推荐简历、匹配分、是否值得投、是否需要微调和主要风险。
- **单岗位分析详情页**：保留过程稿中卡片式分析的气质，适合展示完整分析、打招呼话术、面试建议和微调入口。

## 组件模式

- **ResumeVersionCard**：展示简历版本名称、适用方向、改写重点、最近更新时间和操作入口。
- **JobAnalysisCard**：展示公司、岗位、推荐简历、匹配分、投递建议、微调判断和主要风险。
- **AnalysisDetailSection**：分区展示结论、优势、风险缺口、建议行动、话术和面试建议。
- **CompanyInfoPanel**：承载“公司补充信息，可选”和“联网补充公司信息 Beta / Coming soon”。
- **PrimaryActionBar**：集中承载生成、查看详情、生成微调版、复制、下载等动作。

## 导航与布局

- 页面顶部保留品牌、当前页面标题、主要导航和关键操作。
- 导航集中在顶部或右上区域，不采用左侧竖栏。
- 批量比较页面优先保证扫描效率，单岗位详情页优先保证阅读节奏。
- 重要数字和判断结果应靠前展示，解释内容放在下方或详情区。
- 不要把多个完整岗位分析同时铺满一个页面；总览页只展示决策级摘要。

## 文案风格

- 中文为主，英文只用于品牌、副标签、状态缩写或辅助说明。
- 判断文案要短而明确，例如“推荐”“谨慎”“不推荐”“需要微调”“无需微调”。
- AI 生成内容应结构化呈现，避免单段长文本。
- 占位能力必须明确状态，例如“联网补充公司信息 Beta / Coming soon”。

## 禁止项

- 不做营销式大 Hero。
- 不做聊天机器人中心化首屏。
- 不使用左侧竖栏作为主导航。
- 不使用纯白不透明大卡片覆盖背景。
- 不把所有内容塞进孤立卡片，导致工作台缺少整体结构。
- 不使用过多彩色状态，尤其避免紫色主视觉。
- 不让背景图影响文字和数据可读性。
